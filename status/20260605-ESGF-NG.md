# 2026-06-05 ESGF-NG Status

ESGF-NG status as of June, 2026.

ESGF-NG is the next generation of the core architecture for the Earth System Grid Federation. Designed in 2024, implemented and tested in 2025, and deployed in 2026, it replaces the ESGF-1.0 core architecture. See [ESGF-NG Docs](../docs/README.md) for further information.

**NOTE:** ESGF-NG will be used for CMIP7 and subsequent activities. We plan to begin migrating CMIP6 to ESGF-NG soon, but for the time being, CMIP6, CMIP6plus, and earlier data products will continue using the ESGF-1.0 and ESGF-1.5 systems.

## Monthly Highlights

* **ESGF-NG is ready for use!** There is currently no data in the catalogs because nothing has been published yet.
  * East catalog ([STAC API](https://search.east.esgf.io/), Web app in progress)
  * West catalog ([STAC API](https://search.west.esgf.io/), [Web app](https://metagrid.esgf-west.org/search))
* See the [ESGF-NG onboarding instructions](https://github.com/ESGF/esgf-ng-onboarding/) to obtain permission to publish.
  * Publishing is only available for projects with schemas registered in [ESGVOC](https://github.com/esgf/esgf-vocab).
  * Your data node must be onboarded to an ESGF project before you can publish.
  * ESGF-NG has stricter validation requirements than ESGF-1.0, so you may encounter validation issues with your first publications. We strongly recommend testing your publication in our integration catalog before your first publication to the production catalog.
  * The East and West catalogs are synchronized. Anything published in one will appear in the other automatically.
  * Everything published in the production catalogs will be retained as long as the catalogs remain operational.
* This month, we are working on minor improvements to the ESGF-NG core architecture.
  * A new event stream for tracking successful publication in each ESGF catalog (East and West).
  * Authorization based on data nodes in the West catalog.

## Current and Recent Work

### Publisher Application
The [ESGF Publisher](https://github.com/ESGF/esg-publisher) application (an installable command-line interface tool used by modeling centers) can now be used with the ESGF-NG core architecture. The new features include:
1. Authentication with Globus and EGI Check-in.
2. Publishing with the STAC Transaction API.

Remaining issues are tracked in [this Github project epic](https://github.com/ESGF/esgf-roadmap/issues/95).

### ESGF Replicator Application
We have drafted a technical approach for the ESGF Replicator application, assessing the state of current tools and comparing to known use cases and requirements.  See the [GitHub Repository](https://github.com/ESGF2-us/esgf-replicator) for user stories, requirements, and key design points.

This work will continue in May. Issues are tracked in [this GitHub project epic](https://github.com/ESGF/esgf-roadmap/issues/121).

### Discovery and Data Access Applications
We continue to coordinate with ESGF data access application teams as they test ESGF-NG STAC catalogs (specifically, the STAC discovery API and ESGF project schemas and CVs) and report issues.

This work will continue in June. Issues are tracked in [this GitHub project epic](https://github.com/ESGF/esgf-roadmap/issues/103).

Data access applications include:
* Radiant Earth STAC Browser
* DKRZ’s customisation of the STAC Browser. (Includes support for JupyterLite and visualisations)
* Metagrid
* Intake-esgf
* Esg-pull
* Wget-API
* CMCC
* Climate4Impact
* ESMValTool
* ESGF-py-client - we intend to deprecate this application
